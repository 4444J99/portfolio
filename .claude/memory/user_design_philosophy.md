---
name: User design philosophy — density through layers, not scroll
description: User values minimalism with hidden density, progressive disclosure, attention-holding design. Dislikes vertical scroll, massive cards with sparse text, generic layouts.
type: user
---

The user has a specific design philosophy:

- **"Higher plane of existence"** — wants the portfolio to feel like a premium, crafted experience
- **Density through layers** — all content should exist but be hidden/revealed progressively
- **Anti-scroll** — long vertical pages lose attention. Research confirms 74% of viewing time in first 2 screenfuls
- **"Slick / sexy / sly"** — minimalism that contains weight. Not sparse, not cluttered — layered
- **Every element matters** — when the bento grid removed content, the user demanded it ALL back
- **No massive cards with three lines** — compact, dense cells, not bloated ones
- **Click-to-expand, shift-the-grid** — the user explicitly described the bento interaction pattern they want

The user can SEE when something looks wrong. They review the live site visually and give direct feedback ("it literally looks awful"). Any visual change needs to be reviewed by the user before considering it done.
